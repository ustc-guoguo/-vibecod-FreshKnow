# 鲜知 FreshKnow · AI 冰箱管家与三餐灵感助手

> 打开冰箱，就知道今天吃什么。不浪费，也不将就。

赛道：**生活娱乐 · AI 生活方式**
作品类型：**可交互 Demo（纯前端，单文件 HTML）**
作品状态：**初赛 v1.0**

---

## 一、作品一句话

鲜知不是又一个菜谱搜索，而是从「你家冰箱里现有的食材」出发，**3 秒生成今日三餐 + 未来一周菜单 + 最小补货清单**的 AI 生活方式产品。

它解决的是一个非常真实的日常问题——
**「买了菜，不知道怎么做；打开冰箱，还是不知道今天吃什么。」**

---

## 二、体验链接

- 🌐 **在线体验（推荐）**：`https://<你的域名>/xianzhi-demo/index.html`
- 📦 **离线体验**：下载 `index.html` 双击打开即可，无需任何依赖

> 部署方式见文末「四、如何 1 分钟部署你自己的体验链接」。

体验路径（5 步，约 90 秒）：
1. **组冰箱**：点选你家冰箱里现有的食材（已预选 8 项，含 1 项临期）
2. **设偏好**：人数 / 口味（清淡·减脂·高蛋白·均衡）/ 烹饪时间 / 饮食偏好
3. **AI 生成**：观看一个 3 秒的「鲜知正在为你想今天怎么吃…」过程动画
4. **今日三餐**：早午晚卡片，每餐标注「用到哪些临期食材」、做法、营养、可一键替换
5. **一周菜单 + 补货**：未来 5 天自动排好，最小补货只补 4 样，可勾选

## 三、创新性 / 实用性 / 完成度 / 美观度

### 1. 创新性

| 维度 | 传统菜谱产品 | 鲜知 |
|---|---|---|
| 起点 | 用户先想吃什么，再去搜 | 从冰箱里**已有的东西**出发 |
| 决策 | 把决定权交回给用户 | **AI 直接生成今日方案**，用户只需「采纳 / 替换」 |
| 节奏 | 单餐推荐 | **临期优先 → 今日三餐 → 一周菜单 → 最小补货** 的连续闭环 |
| 浪费 | 不感知保质期 | 内置「临期感」，自动让今天的方案吃掉那盒快坏的菠菜 |

**核心创新点**：把「今天到底吃什么」这件事从一个**搜索问题**转化为一个**约束求解问题**——
约束是你冰箱里现有的食材、保质期、人数、口味、时间、健康目标，目标函数是**少浪费、少决策、好执行**。

### 2. 实用性

- 推理引擎覆盖 25+ 食材、11 道精选做法、5 天菜单、4 项最小补货
- 真正可执行：每道菜都标 `15 分钟`、`一锅出`、`新手友好`
- 替换方案随用：缺西蓝花就给你青椒或蘑菇
- 一键导出：今日方案 + 一周菜单 + 补货清单，复制即可粘贴到备忘录或微信

### 3. 完成度

- ✅ 完整 5 步交互流程，每一步都有**真实数据驱动**的输出（不是静态图）
- ✅ 食材选择 → 偏好 → 三餐生成的**评分推荐算法**（覆盖度 + 临期权重 + 时间权重 + 去重）
- ✅ 「换一组建议」、「换一道午餐」、「重置体验」等微交互全部可用
- ✅ 移动端 / 桌面端均已适配
- ✅ 零外部依赖：单文件 HTML，1390 行，无 React / Vue / Tailwind
- ✅ 支持 4 种快速预设（独居 / 两口之家 / 周末清冰箱 / 清空）

### 4. 美观度 / 设计体验

设计语言：**清新、温暖、克制的生活方式美学**，刻意避开科技蓝赛博风。

- 配色：奶白米白底 + 鼠尾草绿主色 + 番茄红 / 南瓜橙小面积点缀
- 卡片：大圆角、轻阴影、白底，呼吸感充足
- 动效：步骤间淡入、生成动画的脉冲与 ring、卡片 hover 上浮
- 反馈：toast、进度态、勾选态、临期红标，每一处都有视觉反馈

## 四、如何 1 分钟部署你自己的体验链接

Demo 是**零依赖单文件 HTML**，任何静态托管都能直接放上去。

### 方案 A：Cloudflare Pages（推荐 · 永久免费 + 全球 CDN）
1. https://pages.cloudflare.com → Create a project → Direct Upload
2. 把 `xianzhi-demo` 整个文件夹拖进去
3. 立刻拿到 `https://xxx.pages.dev` 永久公开链接，流量无限

### 方案 B：GitHub Pages（永久）
1. 新建仓库，上传 `xianzhi-demo` 目录
2. Settings → Pages → 选择 `main` 分支根目录
3. 几分钟后获得 `https://<user>.github.io/<repo>/xianzhi-demo/`

### 方案 C：Vercel（永久）
```bash
npm i -g vercel
cd xianzhi-demo
vercel --prod
```
得到 `https://xxx.vercel.app`。

### 方案 D：Netlify（必须登录后再拖，否则会过期）
1. 先在 https://app.netlify.com 登录（免费账号即可）
2. 再去 https://app.netlify.com/drop 把 `xianzhi-demo` 文件夹拖进去
3. 站点会绑定到你的账号，永久保留

> ⚠️ **不要用匿名版 Netlify Drop（未登录直接拖）**：那是临时预览链接，约 1 小时后会过期，仅适合本地验证，绝不要作为初赛提交链接。

> 提示：Demo 不调用任何后端 / API，完全本地推理与渲染，因此**部署后无需任何环境变量、密钥、CORS 配置**。

## 五、技术栈

- 纯 **HTML / CSS / JavaScript（ES2020）**
- 无框架，无 npm，无构建步骤
- 模拟 AI 决策的核心算法（约 50 行）：
  - 候选打分：`coverage + urgentBonus(0.5) + warnBonus(0.2) + timeOk(0.1) - duplicate`
  - 临期识别：根据 `status: urgent / warn / ok` 自动加权
  - 一周排盘：去重 + 临期前置 + 补货后才出现的「上新」标记
- 基于 CSS Custom Properties 的设计 token 体系
